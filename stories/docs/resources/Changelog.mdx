# Changelog

All notable changes to BCT UI are documented here. The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.2.0] - 2024-02-17

  - Component Patterns guide
  - Theming guide
  - Accessibility guide
  - Best Practices guide
  - Architecture documentation
  - Contributing guide
- **Token Customizer**: Real-time design token editor
  - Adjust colors, spacing, radius, and shadows
  - Separate light/dark theme customization
  - localStorage persistence
  - Export custom CSS
  - Reset to defaults
- **Enhanced Documentation**: All design token pages now include:
  - Interactive visual examples
  - Copy-to-clipboard functionality
  - Better organization and styling
  - Usage examples with code snippets

- Enhanced color documentation with interactive swatches

## [0.1.0] - 2024-01-20

### Added
- Initial release of BCT UI
- CLI tool for project initialization
- 33 components across 5 categories:
  - Form Inputs (15): Button, TextInput, TextArea, Checkbox, Radio, RadioGroup, Select, Switch, NumberInput, Slider, ColorPicker, DatePicker, DateRangePicker, FileUpload, RichTextInput
  - Feedback (7): Dialog, AlertDialog, Alert, Tooltip, Spinner, Progress, Skeleton
  - Display (6): Badge, Avatar, Card, Breadcrumbs, Pagination, Divider
  - Navigation (4): Accordion, Tabs, DropdownMenu, Popover
  - Utilities (1): Portal
- Design token system with 100+ tokens
- Light and dark theme support
- Tailwind CSS v4 integration
- Base UI primitives for accessibility
- TypeScript support
- Biome for linting and formatting

### Features
- `bct init`: Initialize BCT UI in existing projects
- `bct add`: Add components on-demand
- `bct doctor`: Validate project setup
- Version pinning for compatibility
- Framework support for Vite and Next.js
- Optional i18n with Paraglide
- Optional theme store with Zustand

## Version History

### Versioning Strategy

BCT UI follows [Semantic Versioning](https://semver.org/):

- **Major (X.0.0)**: Breaking changes
- **Minor (0.X.0)**: New features, backward compatible
- **Patch (0.0.X)**: Bug fixes only

### Upgrade Guide

#### From 0.1.x to 0.2.0

No breaking changes. To upgrade:

1. Update CLI: `npm install -g @bctechnology/ui@latest`
2. Update config: Change version in `bct.config.json` to `"0.2.0"`
3. Re-add components: `npx bct add <component-name>`

#### From 0.0.x to 0.1.0

Breaking changes:
- Renamed `Input` to `TextInput`
- Removed deprecated `color` prop from Badge
- Updated design token names for consistency

Migration steps:
1. Update all `Input` imports to `TextInput`
2. Replace `color` prop with `variant` in Badge
3. Update custom token references
