## Step 1: Create or Use Existing Project

### Option A: Create New Vite Project

```bash
pnpm create vite@latest my-app --template react-ts
cd my-app
pnpm install
```

### Option B: Create New Next.js Project

```bash
npx create-next-app@latest my-app
cd my-app
```

### Option C: Use Existing Project

Navigate to your existing Vite or Next.js project:

```bash
cd your-existing-project
```

## Step 2: Install BCT UI CLI

You can install the CLI globally or use it directly with npx:

### Global Installation (Recommended)

```bash
npm install -g @bctechnology/ui
```

### Using npx (No Installation)

```bash
npx @bctechnology/ui init
```

## Step 3: Initialize BCT UI

Run the initialization command in your project directory:

```bash
npx bct init
```

### What Gets Installed

The initialization process will:

- ✅ Copy design tokens to `bct/index.css` or `src/bct/index.css`
- ✅ Install base dependencies (`clsx`, `date-fns`, `@base-ui/react`)
- ✅ Configure Tailwind CSS v4 with BCT tokens
- ✅ Set up Biome for linting and formatting
- ✅ Configure TypeScript path aliases (`@/*`)
- ✅ Create `bct.config.json` with your project settings
- ✅ Install optional features (i18n, theme stores) if selected

## Step 4: Import Design Tokens

Add the BCT design tokens to your main entry file:

### Vite Projects

In `src/main.tsx` or `main.tsx`:

```tsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.tsx'
import './bct/index.css' // Add this line

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
)
```

### Next.js Projects

In `app/layout.tsx`:

```tsx
import './bct/index.css' // Add this line

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

## Troubleshooting

### CLI Not Found

If `bct` command is not found after global installation:

```bash
# Reinstall globally
npm install -g @bctechnology/ui

# Or use npx
npx @bctechnology/ui init
```

### Import Errors

If you see import errors for design tokens:

1. Verify the token file exists at `src/bct/index.css` or `bct/index.css`
2. Check that the import path in your entry file is correct
3. Restart your development server

### TypeScript Errors

If you see TypeScript path alias errors:

1. Verify `tsconfig.json` has the `@/*` path alias configured
2. Restart your TypeScript server in your IDE

### Tailwind Not Working

If Tailwind classes aren't applying:

1. Verify Tailwind CSS v4 is installed
2. Check that `@import "tailwindcss"` is in your token file
3. Restart your development server

## Configuration File

After initialization, you'll have a `bct.config.json` file:

```json
{
  "version": "0.2.0",
  "srcDir": true,
  "features": {
    "i18n": false,
    "themeStore": true
  }
}
```

This file tracks your BCT UI version and configuration. Don't delete it!
