## bct init

Initialize BCT UI in an existing Vite or Next.js project.

### Usage

```bash
npx bct init [options]
```

### Options

      Display help information
    </p>

### Interactive Prompts

The init command will ask you:

1. **i18n Setup**: Do you want to set up internationalization with Paraglide?
2. **Locale Store**: (If i18n enabled) Do you want a Zustand store for locale management?
3. **Theme Store**: Do you want a Zustand store for theme switching?

### What It Does

- ✅ Detects your framework (Vite or Next.js)
- ✅ Detects project structure (src/ directory or not)
- ✅ Copies design tokens to your project
- ✅ Installs base dependencies
- ✅ Configures Tailwind CSS v4
- ✅ Sets up Biome for linting/formatting
- ✅ Configures TypeScript path aliases
- ✅ Creates `bct.config.json`
- ✅ Installs optional features based on your choices

### Example

```bash
# Basic initialization
npx bct init

# Force src directory structure
npx bct init --src-dir true

# Skip src directory
npx bct init --src-dir false
```

### What It Does

1. Reads your `bct.config.json` to get the pinned version
2. Fetches the component from GitHub tag (e.g., `v0.2.0`)
3. Downloads component files to your components directory
4. Installs required npm dependencies
5. Makes the component ready to import and use

### Examples

```bash
# Add a single component
npx bct add button

# Add multiple components
npx bct add button input checkbox

# Add all form components
npx bct add button input textarea checkbox radio select switch
```

### Component Location

Components are placed in:
- `src/components/` if you have a src directory
- `components/` if you don't have a src directory

### Import Path

After adding a component, import it using the `@/` alias:

```tsx
import { Button } from '@/components/button'
import { TextInput } from '@/components/input'
```

## Global vs npx Usage

### Global Installation

Install once, use anywhere:

```bash
npm install -g @bctechnology/ui

# Then use directly
bct init
bct add button
bct doctor
```

### npx Usage

No installation needed:

```bash
npx @bctechnology/ui init
npx @bctechnology/ui add button
npx @bctechnology/ui doctor
```

## Troubleshooting

### Command Not Found

```bash
# If globally installed but not found
npm install -g @bctechnology/ui

# Or use npx
npx @bctechnology/ui <command>
```

### Version Mismatch

If you see version compatibility warnings:

```bash
# Check your pinned version
cat bct.config.json

# Update to latest version
npm install -g @bctechnology/ui@latest
```

### Component Not Found

If a component isn't available:

1. Check the available components list above
2. Verify your BCT version supports that component
3. Try running `bct doctor` to check your setup

## Best Practices

1. **Pin your version**: Don't change the version in `bct.config.json` manually
2. **Run doctor regularly**: Use `bct doctor` to catch issues early
3. **Add components as needed**: Don't add all components at once
4. **Keep CLI updated**: Update the global CLI periodically for bug fixes
