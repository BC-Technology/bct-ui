# Shadows

Shadow tokens add depth and hierarchy to UI elements by simulating elevation. Use shadows to distinguish interactive elements, create visual layers, and guide user attention.

## Usage Examples

```tsx
// Card with medium shadow
<div className="shadow-md rounded-md p-4">
  Card content
</div>

// Button with small shadow on hover
<button className="hover:shadow-sm transition-shadow">
  Hover me
</button>

// Modal with large shadow
<div className="shadow-lg rounded-lg p-6">
  Modal content
</div>
```

## Best Practices

1. **Use sparingly**: Too many shadows create visual clutter
2. **Match elevation to importance**: Higher elevation for more important elements
3. **Combine with transitions**: Animate shadows on hover for better feedback
4. **Consider context**: Shadows work best on light backgrounds
5. **Maintain consistency**: Use the same shadow level for similar elements

## Accessibility

- Shadows should not be the only indicator of interactivity
- Ensure sufficient contrast between shadowed elements and background
- Test shadows in both light and dark themes
