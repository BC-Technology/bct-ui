# Border Radius

Border radius tokens define the corner rounding for UI elements like buttons, cards, and inputs. Consistent corner rounding creates visual harmony across the interface.

## Directional Rounding

Apply radius to specific corners:

```tsx
// Top corners only
<div className="rounded-t-md">Top rounded</div>

// Bottom corners only
<div className="rounded-b-lg">Bottom rounded</div>

// Individual corners
<div className="rounded-tl-md rounded-br-md">Diagonal corners</div>

// Left side
<div className="rounded-l-lg">Left rounded</div>

// Right side
<div className="rounded-r-lg">Right rounded</div>
```

## Best Practices

1. **Be consistent**: Use the same radius for similar elements
2. **Match container radius**: Child elements should have equal or smaller radius
3. **Consider size**: Larger elements can handle larger radius values
4. **Avoid over-rounding**: Too much rounding can look unprofessional
5. **Test responsiveness**: Ensure radius looks good at all screen sizes
