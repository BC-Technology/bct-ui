## What Are Design Tokens?

Design tokens are named entities that store visual design attributes. They're the single source of truth for colors, spacing, typography, and other design decisions. In BCT UI, tokens are implemented as CSS variables that can be used directly in Tailwind classes.

**Example:**
```css
/* Token definition */
--color-primary: #007bff;

/* Usage in Tailwind */
<button className="bg-primary text-primary-on">Click me</button>
```

## Token Categories

### Colors (100+ tokens)
Comprehensive color system including primary, secondary, tertiary, surfaces, typography, status colors, and accents. Each color has variants for hover, focus, and muted states.

**Explore:** [Colors Documentation](/docs/design-system-colors--docs)

### Typography
Text sizes from 10px to 36px, font weights from 100 to 900, and responsive typography classes that adapt to screen size.

**Explore:** [Typography Documentation](/docs/design-system-typography--docs)

### Spacing
Base unit of 0.25rem (4px) with Tailwind's standard spacing scale for consistent layouts.

**Explore:** [Spacing Documentation](/docs/design-system-spacing--docs)

### Border Radius
Four radius values (none, sm, md, lg) for consistent corner rounding across components.

### Shadows
Three shadow levels (sm, md, lg) for elevation and depth effects.

## Implementation

All design tokens are defined in the `index.css` file and are mandatory for all projects using BCT UI components. The tokens are organized using Tailwind CSS v4's `@theme` directive:

```css
@theme {
  --color-primary: var(--color-primary);
  --color-secondary: var(--color-secondary);
  --spacing: 0.25rem;
  --radius-md: 4px;
  /* ... more tokens */
}
```

Theme-specific values are defined in the `@layer base` section:

```css
@layer base {
  [data-theme="light"] {
    --color-primary: #007bff;
    --color-background: #ffffff;
  }
  
  [data-theme="dark"] {
    --color-primary: #4dabf7;
    --color-background: #121212;
  }
}
```

## Usage in Components

Tokens can be used directly in Tailwind classes:

```tsx
// Using color tokens
<div className="bg-primary text-primary-on">Primary button</div>

// Using spacing tokens
<div className="p-4 m-2 gap-3">Spaced content</div>

// Using radius tokens
<div className="rounded-md">Rounded corners</div>

// Using shadow tokens
<div className="shadow-lg">Elevated card</div>
```

## Best Practices

1. **Always use tokens**: Never hardcode colors, spacing, or other design values
2. **Respect semantic meaning**: Use `--color-error` for errors, not just because you like red
3. **Test both themes**: Ensure your UI works in both light and dark modes
4. **Maintain contrast**: Follow accessibility guidelines for text and background contrast
5. **Use the customizer**: Experiment with token values using the interactive customizer
